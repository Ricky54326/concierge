1.1.1 (7 Feb 2013)

New Features

  • Separated Twilio SDK library from OpenSSL and Crypto libraries.
  • Support for armv7(s).

Bug Fixes

  • Fixed an issue where the audio session is dropped upon relaunch after a phone call.
  • Fixed issues which caused crashes in TCDevice and TCConnection objects.
  • Fixed an issue which ceased the creation of new connections.
  • Fixed an issue which prevents accounts with the capability token only allowing for inbound calls to enable inbound calls.
  • Fixed an issue where reachability notifications would over-fire.
  • Fixed an issue where delegate methods wouldn’t be called back in the event of sporadic errors.

1.1.0 (19 Sep 2012)

New Features

  • Increase keepalive interval to bring down device wakeups and reduce idle network usage.
  • Use the Speex wideband codec rather than narrowband, which should result in better voice quality at the expense of slightly greater in-call network usage.
  • Signaling will now recover after switching between WiFi and mobile data (or between different WiFi networks), so if the other party hangs up, the mobile device will receive the notification.
  • The CoreAudio session category is now remembered before a call is placed, and restored after the call terminates.

Bug Fixes

  • Fixed a problem where incoming call notification wouldn’t work if the TCDevice instance was created from a non-main thread.
  • Fixed an intermittent crash that could occur when network reachability status changes.
  • Fixed a possible crash when updating the capability token on an existing TCDevice.
  • Fixed an internal reference cycle that could make it impossible to release a TCDevice instance.
  • Fixed a case where a connection disconnect immediately after a connect could sometimes still allow the call to be placed. In some other cases this would result in the connection delegate’s connection:didFailWithError: method being called with an error reported. This is now just reported as a non-error connectionDidDisconnect:.
